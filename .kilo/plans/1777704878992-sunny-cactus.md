# Plán: Kategoriální editace promptů v knihovně promptů

## Cíl
Rozšířit duální editor (`librarySystemPrompt`, `libraryUserPrompt`) v `promptLibraryModal` tak, aby:
1. **Načítá a ukládá prompty dle aktuálně vybrané kategorie** (tab: Výchozí, Detailní, Stručné, Doslovné, Test, Vlastní)
2. **Ukládá změny persistentně** do `localStorage` pro kategorii `custom` a určitě i pro jiné kategorie (přepisuje built-in preset v rámci session + volitelně persist)
3. **Ochrana před smazáním** – tlačítko "🗑 Vymazat" maže pouze **uživatelsky uložené custom prompty** (storage klíče `strong_custom_system_prompt`, `strong_prompt`, `strong_prompt_library_custom`), nebuilt-in presety
4. **Konzistence UI** – po savu se překreslí seznam a náhled v knihovně, status indicator se aktualizuje

---

## Kontext (z předchozí implementace)

### Soubory a klíče
- **HTML**: `strong_translator.html`, modál `promptLibraryModal` (ř. ~615–668)
- **JS**: `main.js` (funkce `restoreLibraryPrompts`, `saveLibraryPrompts`, `confirmClearLibraryPrompts`), `promptLibrary.js` (api, `showPromptLibraryModal`, `rebuildPromptLibrary`)
- **Storage**:
  - `strong_custom_system_prompt` – systémový prompt (custom)
  - `strong_prompt` + `strong_prompt_mode='custom'` – uživatelský prompt (globální aktuální)
  - `strong_prompt_library_custom` – pole custom entries pro kategorii `custom` (JSON)
  - `strong_prompt_library_imported` – imported entries (přímo nebereme)

### Struktura `state.PROMPT_LIBRARY`
```javascript
{
  default: [{ name, desc, text, system? }],
  detailed: [{ name, desc, text, system? }],
  concise: [{ name, desc, text, system? }],
  literal: [{ name, desc, text, system? }],
  test: [{ name, desc, text, system? }],
  custom: [ ... ] // zahrnuje: 1) uložený custom entry (strong_custom_prompt), 2) imported custom, 3) current prompt, 4) potentially edited built-ins as custom copies
}
```

## Krok 1: Rozšířit strukturu prompt entries o `system`

**Soubor**: `strong_prompts.js` (nebo `aiPromptsResolve.js` pokud se generuje)

Přidat do každé kategorie v `PROMPT_LIBRARY_BASE` pole `system` s výchozí hodnotou `getResolvedSystemMessage()`.

```javascript
default: [{
  name: '...',
  desc: '...',
  text: '...',
  system: getResolvedSystemMessage() // nebo explicitní string
}]
// ... pro detailed, concise, literal, test
```

**Poznámka**: Pokud `PROMPT_LIBRARY_BASE` je statický objekt, protože `getResolvedSystemMessage` není v module `strong_prompts.js` dostupná, můžeme:
- Překládat system prompt z i18n (`aiPrompts.core.system`) přímo v `getResolvedPromptLibraryBase()` (v `aiPromptsResolve.js`)
- Nebo přidat `system` až v `rebuildPromptLibrary()` po volání `getResolvedSystemMessage()` z main.js deps

**Výsledek**: Každý prompt entry v `state.PROMPT_LIBRARY` bude mít `{ name, desc, text, system }`.

## Krok 2: Načítání dle kategorie

**Soubor**: `promptLibrary.js`, funkce `showPromptLibraryModal()` a `selectPrompt()`

1. Při otevření knihovny a při změně kategorie/tab (v `selectPrompt` nebo v onclick handleru tab) načíst do textarea:
   - `librarySystemPrompt.value = state.PROMPT_LIBRARY[category][index].system`
   - `libraryUserPrompt.value = state.PROMPT_LIBRARY[category][index].text`

2. Fallback: pokud entry neobsahuje `system`, použít `getActiveSystemMessage()` (kombinace custom + default).

3. Do existujícího `promptLibraryEditor` (single textarea) i dál načítat `text` (existující chování).

**Implementace** – vytvořit pomocnou funkci:
```javascript
function loadDualEditorForCurrentSelection() {
  const category = state.selectedPromptCategory;
  const index = state.selectedPromptIndex;
  const entry = (state.PROMPT_LIBRARY[category] || [])[index];
  if (!entry) return;
  const sysEl = document.getElementById('librarySystemPrompt');
  const userEl = document.getElementById('libraryUserPrompt');
  if (sysEl) sysEl.value = entry.system || getActiveSystemMessage();
  if (userEl) userEl.value = entry.text;
}
```
Volat:
- Po `rebuildPromptLibrary()` v `showPromptLibraryModal()`
- Na konci `selectPrompt()` (poté, co se nastaví `state.selectedPromptCategory` a `state.selectedPromptIndex`)

## Krok 3: Ukládání dle kategorie

**Soubor**: `main.js`, funkce `saveLibraryPrompts()`

Upravit logiku:
1. Získat `category = state.selectedPromptCategory`, `index = state.selectedPromptIndex`
2. Přepínání podle `category`:
   - **`custom`**:
     - `localStorage.setItem('strong_custom_system_prompt', sysVal)`
     - `setMainPrompt(userVal, 'custom')` (tím se updatuje `strong_prompt` a status)
     - Aktualizovat příslušný entry v `state.PROMPT_LIBRARY.custom`: najít entry podle `index` nebo podle `text` a přepsat `system` a `text`
   - **Built-in kategorie** (`default`, `detailed`, `concise`, `literal`, `test`):
     - **Option A (runtime only)**: Upravit entry v `state.PROMPT_LIBRARY[category][index]` pouze v paměti. Neukládat do localStorage. Při zavření a opětovném otevření se vrátí k base hodnotám (protože `rebuildPromptLibrary()` znovu vytáhne base z `PROMPT_LIBRARY_BASE`). *Tato možnost je jednodušší, ale změny nejsou trvalé.*
     - **Option B (persistent)**: Uložit editaci jako **custom kopii** do `state.PROMPT_LIBRARY.custom` (nový entry s přidanou informací o původní kategorii). Při načítání kategorie `category` zobrazovat editovanou custom kopii, pokud existuje. *Tato možnost je flexibilní a trvalá, ale vyžaduje mapování custom entry -> category.*
     
     **Doporučení**: Začneme s **Option A (runtime only)**, protože je bezpečná a nekomplikuje persistenci. Pokud později bude potřeba trvalost, přidáme Option B.

3. Po updatu entry volat:
   - `rebuildPromptLibrary()` – pokud jsme upravili `custom` kategorii (pro aktualizaci seznamu custom entry)
   - `renderPromptList()` a `renderPromptPreview()` – pro aktualizaci UI
   - `updatePromptStatusIndicator()` – pro aktualizaci status tečky v headeru

4. Zobrazit status "✓ Uloženo" a toast "? AI prompt uložen".

## Krok 4: Reset tlačítka "Obnovit výchozí"

**Soubor**: `main.js`, `restoreLibraryPrompts()`

Upravit chování:
- Pokud kategorie je `custom`: obnovit globální hodnoty (current chování) – `getActiveSystemMessage()` a `getActiveMainPromptTemplate('batch')`.
- Pokud kategorie je built-in: obnovit entry v `state.PROMPT_LIBRARY[category][index]` na výchozí hodnoty z `PROMPT_LIBRARY_BASE` (runtime only). Tedy:
  ```javascript
  const baseEntry = getPromptLibraryBase()[category][index];
  entry.name = baseEntry.name;
  entry.desc = baseEntry.desc;
  entry.text = baseEntry.text;
  entry.system = baseEntry.system; // nebo getResolvedSystemMessage()
  ```
- Nastavit i textarea hodnoty.
- Neukládat do `localStorage` (jen runtime). Pro custom kategorii se zachová stávající chování (update global storage).

---

## Krok 5: Tlačítko "🗑 Vymazat" – vymazat uložené prompty

**Soubor**: `main.js`, `clearLibraryPrompts()` (již existuje)

Upravit `clearLibraryPrompts()`:
1. **Potvrzení** – už máme `confirmClearLibraryPrompts()` s potvrzovacím dialogem.
2. **Smazání storage klíčů**:
   - `localStorage.removeItem('strong_custom_system_prompt')`
   - `localStorage.removeItem('strong_prompt')`
   - `localStorage.removeItem('strong_prompt_mode')` (volitelně, reset na 'system')
   - `localStorage.removeItem('strong_prompt_library_custom')` – smazat custom entries
3. **Reset runtime stavu**:
   - `state.PROMPT_LIBRARY.custom = []` (nebo překonstruovat pomocí `rebuildPromptLibrary()`)
   - Obnovit výchozí entry pro všechny kategorie z `PROMPT_LIBRARY_BASE`
4. **Aktualizace UI**:
   - Pokud je otevřen knihovna, aktualizovat textarea (`librarySystemPrompt`, `libraryUserPrompt`) na výchozí hodnoty
   - Volat `rebuildPromptLibrary()`, `renderPromptList()`, `renderPromptPreview()`, `updatePromptStatusIndicator()`
5. **Toast** – "? Uživatelské prompty byly vymazány"

**Poznámka**: Toto **neovlivní** built-in presety, protože ty jsou v `PROMPT_LIBRARY_BASE` a po clear() se znovu načtou.

## Krok 6: Pořadí inicializace a Live update

### Inicializace editoru při otevření knihovny
Ve `showPromptLibraryModal()`:
1. `rebuildPromptLibrary()` – sestaví `state.PROMPT_LIBRARY`
2. `loadDualEditorForCurrentSelection()` – načte `system` a `text` pro aktuální kategorii a index
3. `renderPromptList()`, `renderPromptPreview()`

### Změna kategorie (tab) nebo výběr jiného promptu
Ve `selectPrompt(index)`:
1. Po nastavení `state.selectedPromptCategory` a `state.selectedPromptIndex`
2. `loadDualEditorForCurrentSelection()`
3. `renderPromptList()` (highlight), `renderPromptPreview()`

## Krok 7: Validace a edge cases

- Pokud je kategorie `custom` a žádný entry neexistuje (prázdné),editor může zobrazit globální custom prompt (pokud existuje) nebo výchozí.
- Při save pro built-in kategorii: pokud `index` je -1 nebo entry neexistuje, neprovádět nic (nebo zobrazit toast "Vyberte prompt").
- Při save pro built-in kategorie: pokud `text` nebo `system` je prázdný, zobrazit error v `libraryPromptStatus` (aktuální chování).
- Při restartu aplikace se built-in edited prompty resetnou na base – to jeAcceptable pro Option A.

## Poznámky k implementaci

- **Použít existující funkce**: `getActiveSystemMessage()`, `getActiveMainPromptTemplate('batch')`, `setMainPrompt()`, `updatePromptStatusIndicator()`, `showToast()`
- **Deps pro promptLibraryApi**: Už máme `getActiveSystemMessage` a `getActiveMainPromptTemplate` přidané – stačí.
- **CSS třída `red`** pro tlačítko Vymazat již v HTML existuje (`.hbtn.red`), použít `class="prompt-btn red"`.
- **Zachovat kompatibilitu** s původním single editorem (`promptLibraryEditor`) – ten slouží pro přesný výběr a editaci jednoho promptu v knihovně.

## Alternativní přístup (Option B persistence)

Pokud bychom chtěli, aby editace built-in kategorií byla **trvalá**, Implementovat:
- Nový storage klíč: `strong_prompt_library_overrides` – mapování `{ category_index: { system, text } }`
- Při load built-in entry: zkontrolovat, zda existuje override; pokud ano, použít ho.
- Při save built-in entry: uložit/zapsat override do storage a také do `state.PROMPT_LIBRARY[category][index]`.

Toto není součástí základního plánu, ale může být přidáno později.

## Dodatečné upřesnění

- **Kdy načítat editor?** Při otevření knihovny a při každé změně vybraného promptu (přepnutí tabu nebo kliknutí na položku v seznamu).
- **Co se stane s custom kategorií?** Pro `custom` zůstává chování unchanged: editor/editace a save kontroluje `state.PROMPT_LIBRARY.custom` – custom kategorie obsahuje mehrere typy entry (vlastní, imported, "current", edited built-ins), ale user bude editovat ** první entry v custom?** nebo **current entry**? Musíme definovat, který index v `custom` kategorii se edituje. Pravděpodobně:
  - Pokud uživatel vybere kategorii `custom`, editor se načte z **prvního** entry (tedy ten, který je právě aktivní/current) – to odpovídá chování `applySelectedPrompt()`, kde se používá `state.PROMPT_LIBRARY.custom[0]` (current). Nebo můžeme edito vždy entry, který je právě vybraný v seznamu (podle `state.selectedPromptIndex`). Pro přehlednost budeme editovat **vždy entry podle `state.selectedPromptIndex`** i v `custom` kategorii (tedy Listing custom entry 0 = current, 1 = imported 1, atd.). Toto je konzistentní.

- **Status message**: pro built-in save zobrazit "✓ Uloženo do kategorie {category}", ale to není nutné, stačí "✓ Uloženo".

## Načtení base prompts s `system` polem

V `rebuildPromptLibrary()`:
`clonePromptLibraryBase()` z `getPromptLibraryBase()` – musí obsahovat `system` pole. Pokud ne, doplníme jej v cyklu:

```javascript
function rebuildPromptLibrary(currentSavedPrompt = '') {
  state.PROMPT_LIBRARY = clonePromptLibraryBase();
  // Doplnit system pole pokud chybí
  for (const cat of ['default','detailed','concise','literal','test']) {
    if (Array.isArray(state.PROMPT_LIBRARY[cat])) {
      for (const entry of state.PROMPT_LIBRARY[cat]) {
        if (!entry.system) {
          entry.system = getResolvedSystemMessage();
        }
      }
    }
  }
  // ... zbytek
}
```

Nebo v `getResolvedPromptLibraryBase()` (aiPromptsResolve.js) rovnou přidat `system` do každé entry.
