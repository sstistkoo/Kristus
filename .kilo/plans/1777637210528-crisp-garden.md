# Plan: Přidat tlačítko pro vymazání local storage překladů

## Cíl
Přidat tlačítko do UI, které vymaže všechny položky localStorage s prefixem `strong_gr_cz_v3_` (překlady), ale zachová API klíče a další nastavení. Přidat upozornění s možností uložit práci před vymazáním.

---

## Part 1: Přidat tlačítko do HTML (strong_translator.html)

- Umístit tlačítko do hlavičky (`hdr`), ideálně vedle existujícího tlačítka `Res.` (clearProgress) nebo v mobile menu.
- Navrhnutí: červené tlačítko s ikonkou 🗑 a textem "Vymazat překlady" / "Clear translations" nebo podobným.
- Zvážit dvě varianty:
  - Variant A: přímo v `hdr-btns` (line ~172-188) – přístupné na desktopu
  - Variant B: pouze v mobile menu (line ~332-345) – nepřidává zatěžování desktop UI
- **Doporučeno**: Variant A, konzistentní s tlačítkem `Res.` (clearProgress) které je také v header.
- ID tlačítka: `btnClearCache` nebo `btnClearTranslations`
- Event handler: `onclick="clearTranslations()"` (nebo `clearCache()`)

**Checkpoint 1**: Ověřit, že tlačítko je viditelné, ma špatně ne kliknutelné, styl odpovídá ostatním `hbtn` tlačítkům (použít třídu `hbtn red` pro destruktivní akci).

---

## Part 2: Vytvořit JavaScript funkci `clearTranslations()`

- Umístění: `js/main.js` (nebo externí modul, ale jednodušší přímo v main.js)
- Funkce musí:
  1. Zobrazit potvrzovací dialog s i18n textem (použít `t()` funkci jako ostatní potvrzení)
  2. Po potvrzení:
     - Vymazat všechny klíče localStorage začínající `strong_gr_cz_v3_`
     - **Zachovat** API klíče (prefix `strong_apikey_`), i18n settings (`strong_ui_lang`, `strong_target_lang`, `strong_source_lang`), a další systémové klíče
     - Resetovat UI stav: vyčistit seznam, log, detaily, statistiky
     - Zobrazit toast s potvrzením "Překlady smazány, API klíče zachovány"
- Reference: Podobně jako `clearProgress()` v `js/backup.js:97-109`, ale nechat API klíče.
- **Důležité**: `state.translated = {};` a překreslit UI.
- Použít pattern:
  ```javascript
  Object.keys(localStorage)
    .filter(k => k.startsWith('strong_gr_cz_v3_'))
    .forEach(k => localStorage.removeItem(k));
  ```

**Checkpoint 2**: Otestovat v konzoli, že po kliknutí se smazají pouze překlady a ne API klíče. Ověřit, že aplikace pak pracuje s prázdným stavem a lze znovu načíst/default data.

---

## Part 3: Přidat i18n klíče do všech jazyků

- Přidat do každého jazykového JSON souboru v `i18n/` (cs.json, en.json, sk.json, atd.):
  - `confirm.clearTranslations`: "Vymazat všechny uložené překlady (localStorage)?\n\nAPI klíče a nastavení zůstanou zachovány."
  - `toast.translationsCleared`: "✅ Překlady smazány, API klíče zachovány"
- Použít stejný styl jako existující `confirm.clearProgress` (line ~72 ve cs.json)
- Pro cs.json vložit okolo existujících message, dodržet formát a členění.

**Checkpoint 3**: Ověřit, že všechny jazyky mají tyto klíče (nebo alespoň cs a en), jinak se zobrazí fallback.

---

## Part 4: (Opcionální) Přidat do Mobile Actions Modal

- Pokud tlačítko není v hlavičce, přidat do `mobileActionsModal` (line ~332-345) jako další položka.
- ID: `btnMobileClearTranslations`
- onclick: `closeMobileModal(); clearTranslations();`
- Styl: `mobl-btn red` (destroy)

## Part 5: Otestovat celou flow

Návod:
1. Otevřít aplikaci, vytvořit nějaké překlady (uložit do localStorage)
2. Otevřít DevTools → Application → Local Storage, ověřit přítomnost keys `strong_gr_cz_v3__...`
3. Kliknout na nové tlačítko
4. Potvrdit dialog
5. Ověřit, že keys `strong_gr_cz_v3_` zmizely
6. Ověřit, že API klíče (např. `strong_apikey_groq`) stále existují
7. Ověřit, že UI se resetovalo (seznam empty, stats 0, log vyčištěn)
8. Otevřít soubor znovu a začít překládat – mělo by fungovat

**Checkpoint 5**: Pokud selže, otevřít konzoli, vyhledat chyby, přepsat.

## Risks
- Nezapomenout na import `t()` funkce v main.js (již tam je, protože clearProgress používá t())
- Nesmazat klíče `strong_gr_cz_v3__backup` nebo `_undo`? Ty jsou podsady, ale po vymazání hlavních dat jsou zbytečné – můžou zůstat nebo být smazányány? Nejlepší smazat i všechny derivované (backup, undo), protože jsou tied to the data.
- Pokud user má několik file slotů (více currentFileId), mazání prefixem `strong_gr_cz_v3_` smaže všechny sloty – což je pravděpodobně požadované chování (všechny překlady).
